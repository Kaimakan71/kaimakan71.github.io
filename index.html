Creating your own operating system

Part I: Introduction

7/4/22 @ 8:30 PM CST
Why make your own operating system? There are plenty of good ones out there already. You probably want your own OS because you think you can make something better than already exists (you can). It's not easy to make an operating system, but you've chosen to do it and I have some experience/code. So, we'll start our journey where the computer starts. You can find this code on my Github repo Kaimakan71/FOSOS. But first, we need a buildsystem.

Part II: The Buildsystem/Prerequisites

The buildsystem is a tool that helps control the way your code is compiled and makes it easier to build. I use make, as it's widely supported and comes preinstalled with MacOS and most Linux distros (Get it here). When you run `make` it looks for a file named `Makefile`. I won't teach you make here, because the goal of this tutorial is to teach you how to make an operating system, not use Make. If you want to learn, there's a tutorial here that seems to go over the core principles you'll need. You should also get a GCC cross-compiler (see GCC cross-compiler) that can produce 32-bit ELF executables, NASM. Another tool we'll need is QEMU, an emulator to run your OS on. This is much easier because real hardware takes a while to start up and costs money to repair if it gets damaged. So, let's start our makefile: 
#### /Makefile ####
# The 'all' target gets run when no other target is specified
all: disk.img

# Build our bootloader
disk.img: boot.asm
	@nasm boot.asm -f bin -o disk.img

# Run QEMU with our disk image
run:
	@qemu-system-i386 -drive format=raw,file=disk.img,if=floppy -m 1M -name NameMe!
Now that you have a buildsystem ready, it's time to make some OS code!
